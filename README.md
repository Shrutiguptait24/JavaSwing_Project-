# JavaSwing_Project-ATM Simulator System
Project Description:
The ATM simulator allowed users to perform basic banking operations such as checking their account balance, withdrawing cash, depositing funds, and changing their PIN. The system was designed to be user-friendly and mimic the interface and functionality of a real ATM.

Technologies Used:

Programming Languages: Java for the core functionality and GUI development.
Database: SQLite for storing user account information and transaction history.
IDE: Eclipse for development and debugging.

My Role:
I was responsible for the entire lifecycle of the project, from gathering requirements and designing the system architecture to coding the application and testing it. I developed the graphical user interface using Java Swing and implemented the backend logic for handling transactions and maintaining database records.

Challenges and Solutions:
One of the major challenges was ensuring the security of user data and transactions. I implemented encryption for sensitive information like PINs and used prepared statements to prevent SQL injection attacks. Additionally, I designed the system to handle concurrent transactions efficiently to avoid any inconsistencies.

Outcomes and Learnings:
The project was successfully completed and well-received by my professors. It provided a comprehensive simulation of ATM operations and was used as a teaching tool for junior students. Through this project, I gained valuable experience in Java programming, GUI development, database management, and software security. It also enhanced my problem-solving skills and ability to manage a project from start to finish.
